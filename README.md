# AutoCG-ML

ML-based auto coarse-graining workflow for MOL2 inputs.
## Prerequisites
Proper mol2 files generated with Chemdraw or Avogardo are recomended.
## Quick Start

```bash
conda env create -f Environment.yml
conda activate autocg_ml
python AutoCG.py -f 1_1.mol2 2_2.mol2 --no-review
bash autorun.sh
```

## Main Outputs

- `output/*.itp`
- `output/*.top`
- `output/*.gro`
- `output/*.pdb`
- `output/*.map`
- `output/forcefield.itp`
- `output/nocharge/*`
NOTE:
1,The bonded parameters of itp files that do not end in '_ML' are generated by atomic cluster centroid algorithm from MOL2 and query of the MAD database. They can be used when users are very confident in the initial structure, but the accuracy of these force fields has not been verified by our development team.
2,All component molecules under the same system should be mapped in the same batch.
## Repo Notes

- `AutoCG.py`: mapping + topology generation.
- `scripts/`: ML prediction and force-field assembly steps.
- `autorun.sh`: end-to-end post-processing pipeline.
